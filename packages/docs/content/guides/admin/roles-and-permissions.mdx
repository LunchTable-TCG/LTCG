# Admin Roles and Permissions

The LTCG admin system uses a hierarchical role-based access control system.

## Role Hierarchy

The system uses these roles in descending order of privileges:

### 1. Superadmin (Highest Privileges)
- Can grant/revoke all roles including other superadmins
- Full system access
- Can manage all aspects of the platform

### 2. Admin
- Manage users, content, economy
- Cannot manage other admins
- Standard administrative capabilities

### 3. Moderator
- Content moderation
- User management (limited)
- Cannot access economy or treasury

### 4. Support
- Read-only access
- Ticket management
- Cannot modify data

### 5. Viewer
- Read-only dashboard access
- Cannot perform any actions

## Promoting Users

Use the admin dashboard or Convex functions to promote users:

### Promote to Admin

```bash
npx convex run admin/roles:grantRole \
  --targetUserId "USER_ID" \
  --role "admin" --prod
```

### Promote to Superadmin

Requires an existing superadmin to execute:

```bash
npx convex run admin/roles:grantRole \
  --targetUserId "USER_ID" \
  --role "superadmin" --prod
```

## Permission Matrix

| Action | Superadmin | Admin | Moderator | Support | Viewer |
|--------|------------|-------|-----------|---------|--------|
| Grant/Revoke Superadmin | ✅ | ❌ | ❌ | ❌ | ❌ |
| Grant/Revoke Admin | ✅ | ❌ | ❌ | ❌ | ❌ |
| Manage Users | ✅ | ✅ | Limited | ❌ | ❌ |
| Manage Content | ✅ | ✅ | ✅ | ❌ | ❌ |
| Manage Economy | ✅ | ✅ | ❌ | ❌ | ❌ |
| Manage Treasury | ✅ | ✅ | ❌ | ❌ | ❌ |
| View Dashboard | ✅ | ✅ | ✅ | ✅ | ✅ |
