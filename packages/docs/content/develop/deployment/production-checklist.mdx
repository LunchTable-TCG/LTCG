# Production Deployment Checklist

Complete checklist for deploying LTCG to production.

## Pre-Deployment Checklist

Ensure all these items are complete before deploying:

- [ ] Environment variables configured on Vercel/hosting
- [ ] Convex production deployment created
- [ ] Privy production app configured
- [ ] Solana mainnet RPC configured (if using treasury)
- [ ] Run complete setup on production:
  ```bash
  convex run setup:setupComplete --prod
  ```
- [ ] Create superadmin on production:
  ```bash
  convex run setup:setupSuperadmin \
    --privyUserId "YOUR_PROD_PRIVY_ID" --prod
  ```
- [ ] Verify setup status:
  ```bash
  convex run setup:checkSetupStatus --prod
  ```

## Environment Variables

Required for production deployment:

### Convex
```env
CONVEX_DEPLOYMENT=       # Your production deployment name
NEXT_PUBLIC_CONVEX_URL=  # Your production Convex URL
```

### Privy Authentication
```env
NEXT_PUBLIC_PRIVY_APP_ID=  # Production Privy app ID
PRIVY_APP_SECRET=          # Production Privy app secret
PRIVY_APP_ID=              # Production Privy app ID
```

### Solana (Optional)
```env
NEXT_PUBLIC_SOLANA_RPC=    # Mainnet RPC endpoint
NEXT_PUBLIC_SOLANA_NETWORK=mainnet-beta
```

### Token Configuration
```env
NEXT_PUBLIC_LTCG_TOKEN_MINT=      # Production token mint address
NEXT_PUBLIC_LTCG_TOKEN_DECIMALS=6
```

## Post-Deployment

After deployment is complete:

1. Test admin login on production domain
2. Verify all features work
3. Set up monitoring/alerts
4. Document admin credentials securely
5. Configure backup strategy
6. Set up error tracking (Sentry, etc.)

## Deployment Configuration

### Vercel Settings

**Web App:**
- Root Directory: `apps/web`
- Build Command: `bun run build:web`
- Output Directory: `.next`

**Admin App:**
- Root Directory: `apps/admin`
- Build Command: `bun run build:admin`
- Output Directory: `.next`

## Rollback Plan

If issues occur after deployment:

1. Revert to previous Vercel deployment
2. Check Convex logs: `npx convex logs --prod`
3. Verify environment variables
4. Contact support if needed

## Monitoring

Set up monitoring for:
- Application errors
- API response times
- Database query performance
- User authentication failures
- Treasury operations
