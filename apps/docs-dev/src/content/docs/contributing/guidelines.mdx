## Welcome!

Thank you for your interest in contributing to LunchTable TCG. This guide will help you get started.

## Getting Started

<Steps>

1. **Fork the repository**

   Click "Fork" on the [LunchTable TCG GitHub page](https://github.com/ltcg/ltcg).

2. **Clone your fork**

   ```bash
   git clone https://github.com/YOUR_USERNAME/ltcg.git
   cd ltcg
   ```

3. **Set up the development environment**

   See [Installation Guide](/quick-start/installation/).

4. **Create a feature branch**

   ```bash
   git checkout -b feature/your-feature-name
   ```

5. **Make your changes**

   Follow the [Code Style Guide](/contributing/code-style/).

6. **Submit a Pull Request**

   See [Pull Request Guide](/contributing/pull-requests/).

</Steps>

## What to Contribute

### Good First Issues

Look for issues labeled `good-first-issue` on GitHub. These are:
- Well-defined scope
- Documented requirements
- Mentorship available

### Feature Development

Before starting a new feature:
1. Check existing issues and PRs
2. Open a discussion or issue first
3. Wait for maintainer approval
4. Follow the design process

### Bug Fixes

1. Reproduce the bug locally
2. Write a failing test
3. Fix the bug
4. Ensure all tests pass

### Documentation

- Fix typos and unclear sections
- Add missing documentation
- Improve code examples
- Translate documentation

## Code of Conduct

### Expected Behavior

- Be respectful and inclusive
- Welcome newcomers
- Assume good intent
- Focus on constructive feedback

### Unacceptable Behavior

- Harassment or discrimination
- Trolling or personal attacks
- Publishing private information
- Other unprofessional conduct

<Aside type="note">
  Report violations to maintainers via GitHub or Discord DM.
</Aside>

## Development Workflow

### Branch Naming

| Type | Format | Example |
|------|--------|---------|
| Feature | `feature/description` | `feature/add-draft-mode` |
| Bug fix | `fix/description` | `fix/combat-damage-calc` |
| Docs | `docs/description` | `docs/api-reference` |
| Refactor | `refactor/description` | `refactor/game-engine` |

### Commit Messages

We follow [Conventional Commits](https://conventionalcommits.org):

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

**Types:** `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

**Examples:**

```bash
feat(cards): add poison ability type
fix(combat): correct element damage modifier
docs(api): add economy API reference
refactor(engine): extract effect resolver
```

## Getting Help

- **Documentation:** Browse these docs
- **Discord:** Join #dev-help channel
- **GitHub:** Open a discussion
- **Issues:** Search existing issues
